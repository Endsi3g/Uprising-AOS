Stack 2026 — Mis à Jour
Upgrade recommandé : Les recherches 2026 confirment que Next.js 16 + React 19 + Tailwind v4 est maintenant le standard de production pour les dashboards admin avec shadcn/ui. La colocation-based architecture (feature keeps its own pages/components) est le pattern dominant. On upgrade le blueprint en conséquence — le changement de Next.js 15 → 16 est non-breaking.
Frontend Core
Next.js 16 + React 19
App Router, React Compiler (auto-memoization), Server Components, Server Actions. Colocation-first architecture — chaque module garde ses pages/components/logic dans son dossier.

Agency-Agents — 14 Agents Sélectionnés pour Uprising OS
C'est quoi ce repo ?
msitarzewski/agency-agents (86.9k ⭐) est une collection de 144 agents IA spécialisés avec personnalité, workflows et livrables concrets. Chaque agent est un fichier .md installé dans ~/.claude/agents/ pour Claude Code. Tu les actives en disant "Use the [Agent Name] agent" dans ta session Claude Code. Pour Uprising Studio, j'ai sélectionné les 14 agents les plus pertinents parmi les divisions Marketing, Sales, Engineering, Design et Project Management.
Comment les installer
Clone le repo, run le script d'installation pour Claude Code. Les agents s'installent automatiquement dans ~/.claude/agents/. Pendant le dev de l'Agency OS, tu peux activer le Frontend Developer pour coder, l'Instagram Curator pour le module Contenu, l'Outbound Strategist pour le module Prospection, etc. Ils connaissent leur domaine en profondeur et produisent du code + des livrables directs.
Terminal — Installation Claude Code
# 1. Clone le repo
git clone https://github.com/msitarzewski/agency-agents
cd agency-agents

# 2. Install pour Claude Code (tous les agents)
./scripts/install.sh --tool claude-code

# 3. Ou copier seulement les agents sélectionnés pour Uprising
cp marketing/marketing-instagram-curator.md ~/.claude/agents/
cp marketing/marketing-content-creator.md ~/.claude/agents/

Structure de Fichiers — Colocation Architecture 2026
Pattern recommandé 2026 : chaque feature garde ses propres pages, components et logic dans son dossier de route. Les composants partagés et la config restent au top level. Ce pattern est plus scalable et co-loc friendly pour les agents Claude Code.
